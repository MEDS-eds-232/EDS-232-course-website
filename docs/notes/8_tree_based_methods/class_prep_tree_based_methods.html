Regression trees

This is a very brief section of an example of what a regression tree is with a diagram. It includes:

  • a diagram of a regression tree.
  • description of what nodes, leaves, branches, and root are.
  • an example of how an observation gets assigned a prediction.

The goal is to have students understand how to follow along the branches of a decision tree to for a regression problem.

We also explain that the value in the leaf is the average of the values for all the observations in the training set that satisfy all the conditions in the nodes that were taken to get to the leaf.

Check-in: use the regression tree to obtain predictions for a given observation.

Fitting regression trees

This is a short section where we build one decision tree with only one root. It includes showing the decision boundary for that one node as a boundary on X_1.

Then we move the decision boundary by one unit and recalculate the one-node regression tree.

Then we add one node by doing another subset of the predictor space now on the second variable X_2.

Check-in: Have students draw the splits in predictor space for a given three-node decision tree and fill in the values for the leaves.

So a regression tree works by:

  1. Dividing predictor space (this is the set of all possible values for \(X_1, X_2, \ldots, X_p\)) into non-overlapping regions \(R_1, \ldots, R_J\).

  2. For every observation that falls into a region \(R_j\), we assign the same prediction, which is the mean of the response values for the training observations in \(R_j\).

To fit the regression tree so it makes the best possible predictions, we need to establish what the splitting rules are. Equivalently, we need to find a way to divide the predictor space into regions that give good enough approximations.

The process by which these splits are found is called recursive binary splitting. Essentially, after each split, this algorithm looks to minimize an error term capturing the difference between the average value for the observations in a given region, and the actual response values.

Dealing with overfitting

Deep decision trees are prone to overfitting with decision boundaries that are interpretable but not stable, and small changes in the training data leading to big difference in outcome (high variance).

The core idea is that, instead of using the full tree, we can overcome overfitting by using a subtree withing the tree instead.

Some ways to get a subtree are:

  1. specifying the maximum depth of the tree
  2. specifying the minimum number of observations to be at a leaf node
  3. cost complexity pruning: balance complexity and fit to training data

In options 1 and 2 we use the maximum depth and the minimum number of observations at a leaf as hyperparameters for the model. Cost complexity pruning also uses a hyperparameter that controls the weight given to the tree’s complexity.

Example: Show examples of pruned trees coming from the full tree.

Classification trees

All of the classification trees section uses a very small dataset with two numerical predictors X_1, X_2, and one categorical response variable with three classes that we can follow along. Maybe some 20 observations.

Start with a plot of the data in X_1, X_2 space with each point colored by its class.

Decision trees can also be used for classification problems, in this case we call them classification trees.

Check-in: A small classification tree is given and the exercise asks them to find a classification for a new observation.

For a classification tree, we predict that each observation belongs to the most commonly occurring class of training observations in the region to which it belongs.

Check-in: A diagram with the decision boundaries on the data plot and students are asked to make a decision tree from it.

We are often interested not only in the class prediction corresponding to a particular terminal node region, but also in the class proportions among the training observations that fall into that region.

Fitting a classification tree

So a classification tree works by:

  1. Dividing predictor space (this is the set of all possible values for \(X_1, X_2, \ldots, X_p\)) into non-overlapping regions \(R_1, \ldots, R_J\).

  2. For every observation that falls into a region \(R_j\), we assign the same prediction, which is the majority class for the training observations in \(R_j\).

As before, to fit the classification tree so it makes the best possible predictions, we need to establish what the splitting rules are. Equivalently, we need to find a way to divide the predictor space into regions that give good enough predictions.

As before, we recursive binary splitting is used to search for the best split at each step.
