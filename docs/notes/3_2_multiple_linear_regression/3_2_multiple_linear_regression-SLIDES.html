The multiple linear regression model

From SLR to MLR

In SLR we modelled the relationship between one predictor and a response:

\[\texttt{co2\_reduction} \approx \hat{\beta}_0 + \hat{\beta}_1 \cdot \texttt{process}\]


But the GreenAbate dataset has three potential predictors. Running separate SLRs has two problems:

  • No single prediction combining all three predictors
  • Each SLR ignores the other predictors — coefficients can be misleading when predictors are correlated


Multiple linear regression handles all predictors simultaneously.

The MLR model

With \(p\) predictors \(X_1, \ldots, X_p\) and response \(Y\):

\[Y = \beta_0 + \beta_1 X_1 + \beta_2 X_2 + \cdots + \beta_p X_p + \epsilon\]


The key phrase in interpreting each coefficient:

\(\hat{\beta}_j\) is the average change in \(Y\) associated with a one-unit increase in \(X_j\), holding all other predictors constant.


For GreenAbate (\(p = 3\)):

\[\texttt{co2\_reduction} = \beta_0 + \beta_1 \cdot \texttt{process} + \beta_2 \cdot \texttt{efficiency} + \beta_3 \cdot \texttt{offsets} + \epsilon\]

Fitted MLR coefficients for GreenAbate

 Intercept      3.4543     0.3416       10.11 < 0.0001
   process      0.0461     0.0013       35.37 < 0.0001
efficiency      0.1831     0.0075       24.37 < 0.0001

The fitted model is:

\[\widehat{\texttt{co2}} = 3\.454 + 0\.0461 \cdot \texttt{process} + 0\.1831 \cdot \texttt{efficiency} \-0\.0088 \cdot \texttt{offsets}\]

p-values for the coefficients

Individual \(p\)-values: partial effects

In SLR, the \(p\)-value for \(\hat{\beta}_1\) tests \(H_0: \beta_1 = 0\) — is \(X\) related to \(Y\)?


In MLR, the \(p\)-value for each \(\hat{\beta}_j\) tests the partial effect of \(X_j\):

Does \(X_j\) add information beyond what the other predictors already explain?


  • Small \(p\)-value for \(\hat{\beta}_j\) → \(X_j\) is informative given the other predictors are in the model
  • Large \(p\)-value for \(\hat{\beta}_j\) → \(X_j\) adds little beyond what the others already capture

Check-in

The offsets variable has a p-value of 0.0193 in the MLR.

Does this mean carbon offset purchases have no relationship with CO₂ reductions?

What exactly does this p-value tell us?

Check-in — answers

Not necessarily. A large p-value means offsets adds little beyond what process and efficiency already explain.

In a separate SLR of co2_reduction on offsets alone, the result could differ.

Here, the true coefficient for offsets is \(-0.001\) (near zero), so the model is correctly detecting it contributes almost nothing.

Comparing MLR to individual SLRs

SLR vs. MLR: why not run separate regressions?

 Predictor SLR coeff SLR p-value MLR coeff MLR p-value
   process    0.0460    < 0.0001    0.0461    < 0.0001
efficiency    0.1793    < 0.0001    0.1831    < 0.0001
   offsets   -0.0060      0.6153   -0.0088      0.0193


  • In this dataset, SLR ≈ MLR coefficients — because the predictors were drawn independently (uncorrelated by construction).
  • In real data, predictors are typically correlated. The divergence can be large, sometimes reversing the sign of a coefficient.
  • The MLR coefficient reflects the direct effect of each predictor, holding the others constant.

Hypothesis testing: the F-statistic

Is any predictor related to the response?

In SLR we tested \(H_0: \beta_1 = 0\). In MLR, the overall question is:

\[H_0: \beta_1 = \beta_2 = \cdots = \beta_p = 0\] \[H_a: \text{at least one } \beta_j \neq 0\]


We test this with the \(F\)-statistic:

\[F = \frac{(TSS - RSS)/p}{RSS/(n - p - 1)}\]

  • \(F \approx 1\): no predictor is related to \(Y\) (consistent with \(H_0\))
  • \(F \gg 1\): at least one predictor explains variance beyond noise → evidence against \(H_0\)

F-statistic for the GreenAbate MLR

   Quantity    Value
F-statistic   612.13
    p-value < 0.0001


An \(F\)-statistic of 612.1 with \(p \ll 0.05\) gives very strong evidence that at least one predictor is associated with CO₂ reductions.


Why not just use the individual p-values?

When \(p\) is large, some individual p-values will be small by chance even when no predictor is truly related to \(Y\). With \(p = 100\) truly null predictors, we expect ~5 to have \(p < 0.05\) by random chance alone. The \(F\)-statistic tests all coefficients jointly and accounts for this.

Model fit: R² in MLR

\(R^2\) in multiple regression

The \(R^2\) formula is the same as in SLR:

\[R^2 = 1 - \frac{RSS}{TSS}\]


Important caveat: adding a predictor to an MLR model will always increase \(R^2\) (or leave it unchanged), even if that predictor has no real relationship with \(Y\).


The adjusted \(R^2\) penalizes for the number of predictors:

\[\bar{R}^2 = 1 - \frac{RSS/(n-p-1)}{TSS/(n-1)}\]

Adjusted \(R^2\) can decrease when an irrelevant predictor is added — making it better for comparing models of different sizes.

Comparing SLR and MLR model fit

                     Model    R² RSE (ktCO₂e)
        SLR (process only) 0.611        3.138
MLR (all three predictors) 0.904        1.571


  • \(R^2\) increased from 0.611 to 0.904 — efficiency is genuinely predictive.
  • RSE decreased from 3.138 to 1.571 ktCO₂e — predictions are more accurate with all three predictors.

Variable selection

Which predictors should we include?

With \(p\) predictors there are \(2^p\) possible subsets:

  • \(p = 3\): 8 models — feasible to compare exhaustively
  • \(p = 10\): 1,024 models
  • \(p = 30\): over 1 billion models — completely infeasible


Three common automated approaches for large \(p\):

  • Forward selection: start with nothing, add one predictor at a time
  • Backward selection: start with everything, remove one at a time
  • Mixed selection: combine forward and backward steps

Forward and backward selection

Forward selection

  1. Start: null model (intercept only)
  2. Add the predictor with the largest RSS reduction
  3. Repeat until stopping criterion is met

✓ Always applicable (works when \(p > n\))

✗ A variable added early may become redundant later

Backward selection

  1. Start: full model (all \(p\) predictors)
  2. Remove the predictor with the largest \(p\)-value
  3. Repeat until all remaining predictors are significant

✓ Considers all predictors from the start

✗ Cannot be used when \(p > n\)

Mixed selection

Mixed selection (stepwise) combines forward and backward steps:


  1. Start with the null model
  2. Forward step: add the variable with the largest RSS reduction
  3. Backward step: check if any included variable now has a large \(p\)-value — if so, remove it
  4. Continue until no variables are added or removed


This fixes the main weakness of forward selection: a variable added early can be removed once other, better variables are in the model.

Other criteria for variable selection

RSS alone always favors larger models — use criteria that penalize complexity:


Criterion Formula Goal
Adjusted \(R^2\) \(1 - \frac{RSS/(n-p-1)}{TSS/(n-1)}\) Maximize
AIC \(n\log(RSS/n) + 2p\) Minimize
BIC \(n\log(RSS/n) + p\log(n)\) Minimize
Test MSE (CV) held-out set error Minimize


BIC penalizes extra predictors more heavily than AIC (for \(n > 7\)), tending to select sparser models. Cross-validation is the most direct approach when the goal is prediction accuracy.
